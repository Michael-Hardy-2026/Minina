## (zh-hans) 简体中文

> Minina：一个极致精简、开放生长的指令集架构

**项目缘起**

Minina 源于一个简单的信念：开源硬件的发展，需要更多元的底层选择和更健康的社区生态。我们见证了 *RISC-V* 的伟大成功，也认识到任何单一技术路径在快速发展中可能面临的挑战。

Minina 并非为“替代”而生，而是为“探索”而设计。我们旨在构建一个核心极度稳定、边界彻底开放的指令集架构（ISA），通过制度化的开放创新空间，激励社区在最底层进行协作与实验，共同应对碎片化、生态发展等系统性难题，为全球开源硬件生态注入新的活力。

**核心设计亮点**

- 双模指令集，快速启航：基础指令集（标准模式）兼容成熟的 RISC-V 六种格式，可无缝复用现有编译器与工具链生态，极大降低入门门槛。同时提供高密度压缩指令模式，满足嵌入式场景的严苛需求。
- 结构化开放边界，赋能社区：我们首创了 “社区实验指令空间” 。任何人都可以遵循简易流程，申请一块独属于自己项目的指令编码空间，用于实现领域专用加速器（DSA）、学术研究或任何天马行空的创新，并得到官方工具链的原生支持。
- 渐进式标准化路径：在“实验空间”中经社区验证的优秀扩展，可通过公开评审流程，晋升为官方标准扩展。这为社区创新提供了明确的上升通道，确保架构能在稳定核心的基础上持续进化。
- 现代、精简的起点：从零设计，无历史包袱。我们汲取了现有 ISA 的设计智慧，力求在指令格式和编码上做到简洁、正交，为高性能、低功耗的实现奠定坚实基础。

**技术架构概览**

Minina 的指令编码基于一个清晰的三层模型，由指令的最低有效位（inst[1:0]）定义：

- inst[1:0]模式描述01标准模式32位基础指令，格式与RISC-V的R/I/S/B/U/J型兼容，构成稳定核心。
- 00 或 10压缩模式16位高密度指令，用于提升代码密度，支持常用操作。
- 11扩展空间通往开放创新的大门。此模式下的指令用于各类扩展。

**社区实验空间：您的创新沙盒**

在扩展空间（inst[1:0]=11）内，我们专门划出了 “社区实验区”（当 inst[3:2]=11 时）。这是 Minina 哲学的核心体现：

- 自由申请：任何个人或团队均可通过提交 Pull Request 到本仓库的 extensions/ 目录，申请一个唯一的 实验ID（编码于 inst[7:4]）。
- 自主定义：申请通过后，您将获得对应 实验ID 下所有指令编码的完全定义权，用于实现您的专属扩展。
- 生态集成：我们维护的基础工具链（模拟器、汇编器）将支持动态加载您的扩展插件，让创新能立刻被验证和分享。

我们相信，真正的创新来自边缘。Minina 致力于将这种边缘创新制度化、主流化。

**快速开始**

请注意：Minina 目前处于早期开发阶段，热烈欢迎早期建设者！

- 阅读文档：深入了解**核心指令集规范**和**扩展申请指南**。
- 获取工具：使用我们早期的 Minina 工具链 （包含模拟器和汇编器）进行体验和开发。
- 提出想法：在 GitHub Issues 中讨论架构设计或提出您的扩展构想。
- 成为拓展者：按照指南，提交您的第一个扩展申请，成为 Minina 生态的创始贡献者！

**项目路线图**

阶段 1：概念验证（当前） - 确定基础指令格式，完成最小可行模拟器。

阶段 2：生态筑基 - 稳定核心指令集，完善基础工具链（GCC/LLVM端口），建立扩展管理流程。

阶段 3：社区繁荣 - 推动首批社区扩展项目，在 FPGA 或模拟环境中展示多样化应用。

阶段 4：硬件实践 - 与合作伙伴共同推动开源芯片实现，走向硅验证。

**参与贡献**

Minina 的成功完全依赖于一个活跃、开放的社区。我们欢迎各种形式的贡献：

- 架构设计：参与指令集和系统架构的讨论。
- 软件开发：贡献于模拟器、编译器、调试器、操作系统移植等。
- 扩展创新：申请您的实验空间，制作并分享一个炫酷的扩展。
- 文档与布道：帮助完善文档，撰写教程，在任何地方分享 Minina 的理念。

请参阅[贡献者指南](.github/CONTRIBUTING.md)了解如何开始。

**为什么叫 “Minina”？**

名字源自 “迷你” 与 “纳” 的组合，寓意着在追求核心精简（Mini）的同时，拥抱底层无限小、无限多的开放可能性（Nano）。
